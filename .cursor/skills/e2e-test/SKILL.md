# E2E Test — 端到端测试

通用的 E2E 测试 skill，通过浏览器自动化验证前后端联动行为。

## 前置条件

- 前后端服务已启动
- 浏览器自动化工具可用（Playwright MCP 或 agent-browser CLI）

## 执行流程

### Step 1: 环境检查

依次检查，任何一项失败都停下报告：

1. **浏览器工具可用性**：尝试访问一个简单页面
2. **后端服务**：检查健康检查接口
3. **前端服务**：浏览器打开首页确认加载
4. **登录状态**：如需登录，执行登录操作

全部通过后输出环境检查结果。

### Step 2: 设计测试用例

基于需求产出物设计测试用例：

1. 读取 `requirements/<req-id>/` 下的 brainstorm.md、plan.md、implementation.md
2. 识别涉及的页面、API、业务流程
3. 设计测试用例表：

| ID | 优先级 | 用例名称 | 操作步骤 | 期望结果 |
|----|--------|---------|---------|---------|
| TC-001 | P0 | ... | ... | ... |

### Step 3: 执行测试

按用例表逐个执行：

1. 导航到目标页面
2. 按步骤操作（点击、输入、提交等）
3. 验证期望结果（页面状态、Toast 提示、数据变化）
4. 截图记录关键状态
5. 记录结果：PASS / FAIL

失败处理：
- 单个用例失败不阻塞后续执行
- 记录失败截图和实际页面状态
- 连续 3 个因环境问题失败则停下报告

### Step 4: 输出测试报告

**产出**：`requirements/<req-id>/e2e-test.md`

```markdown
## E2E 测试报告

### 环境信息
- 后端地址: <地址>
- 前端地址: <地址>
- 执行时间: <时间>

### 执行结果汇总
| 统计项 | 数量 |
|--------|------|
| 总用例数 | X |
| PASS | Y |
| FAIL | Z |
| SKIP | W |

### 用例明细
| ID | 用例名称 | 结果 | 备注 |
|----|---------|------|------|

### 失败用例详情
#### FAIL TC-XXX: <用例名称>
- 期望: ...
- 实际: ...
- 截图: ...
- 可能原因: ...
```

## 约束

- **不修改业务代码**：发现 Bug 只记录，不修复
- **不跳过环境检查**：任何检查失败都必须停下
- **截图留证**：每个失败用例必须截图
