## 原始需求

问题发送出去显示是好的，但一旦接收到后端响应，前端会丢失当前显示内容。需要把“创建新树 + 发送首个问题 + 生成首个回答”重构成流式编排，避免前端在响应返回后清空状态。

## 需求摘要

这个 feature 要把新树创建流程改成流式、渐进式的用户体验，而不是依赖一次性后端返回结果。用户进入新树页面后先输入第一个问题，提交后系统在同一条流程里完成：

1. 创建新树
2. 创建 root node
3. 创建首个 `qapair`
4. 逐步返回回答内容

前端需要在这个过程中保留同一条消息的展示状态，不能在后端响应回来后把页面清空或切换成空状态。

## 已确认方向

- 新树创建页继续保留为独立入口
- 首问提交采用流式输出，而不是一次性完整响应
- 后端需要支持“创建树 + 首答生成”的编排
- 前端需要支持增量更新同一条回答，不依赖整页刷新
- 失败时保留当前页面状态，允许重试，不应丢失已输入问题
